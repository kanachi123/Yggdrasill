 # ***Ternary tree***

 ## **Троичное дерево это такая древовидная структура данных в котором каждый узел может иметь максимум три потомка левый,средний,правый**

 ```
       [---------------------------]
        |              |          |
        v              v          v
     [-----]         [--]        [-]
    /   |  \   [-]<-/   \->[-]     \->[-]
    |   |   |
    v   v   v
   [-] [-] [-]        
```

### **Для реализации используются тернарные деревья:**
*Тернарные деревья поиска и Троичные кучи*

## **можно посчитать максимальное количество узлов дерева зная высоту тернарного дерева**

поскольку корневой узел всегда один будем это учитывать,
для последующих узлов будет потомок count[0,3]
**если h это высота дерева,тогда M(h) = 1(сам главный корень) + ... $3^h$ = $\sum_{i = 0}^{h} 3^i$ = $({3^{h+1} - 1})/{2}$**

по комбинаторике для каждой ноды имеем три выбора по высоте
то есть 
h = 1 
нода имеет три возможных выбора максимум
отсюда 3 ,
но каждый уровень увеличивает количество i раз
то есть $3^i$

эту последовательность можно считать геомтерической прогресией,а значит по формуле геометрической прогрессии 
${(q^{n+1}-1)}/{q-1}$ можем посчитать сумму $({3^{h+1} - 1})/{2}$

причина в том что в iой глубине появляется i вершин с 3 выборами получается $3^i$
то есть i родителей,но каждый по 3 ребенка максимум получается $3^i$

Обработка текстов и строк:
Автоматизация поиска: Тернарные деревья используются в алгоритмах для эффективного поиска подстрок и обработки строк. Например, в алгоритмах поиска шаблонов, где строка разбивается на три категории (меньше, равно и больше) для быстрого поиска.
Бинарные деревья с улучшением производительности:
В некоторых алгоритмах или модификациях бинарных деревьев тернарная структура может быть полезной для балансировки и более эффективного поиска, если каждый узел хранит дополнительные данные для более сложных операций.
Представление многоклассовых решений:
В задачах классификации, например, в машинном обучении, когда необходимо классифицировать объекты в три категории, тернарные деревья могут быть использованы для представления таких решений.
Поиск и сортировка:
В случаях, когда данные требуют представления и поиска в более сложных многоклассовых структурах, например, при организации многоклассовых очередей или обработки данных с трёхуровневыми решениями.
Иерархические структуры:
В некоторых задачах, связанных с организацией иерархий, где каждый узел может иметь до трёх дочерних элементов, тернарное дерево помогает эффективно организовывать данные (например, в некоторых вариациях файловых систем или баз данных).
Алгоритмы с трёхуровневыми решениями (например, с тремя состояниями):
В системах с трёхуровневыми состояниями или с возможностью тройной оптимизации тернарные деревья могут представлять собой более естественное решение.



## арифметика Лукасевича и nullable в C#

у тебя три выбора
правда
ложь
неизвестно

можно так использовать например в играх где персонаж играет через ложную дихотомию

В языке C# можно использовать nullable bool (bool?), чтобы хранить три состояния: True, False и null (неизвестно)


```C#

using System;

class Program
{
    static void Main()
    {
        bool? decision = GetPlayerDecision(); // Получаем выбор игрока (неизвестно, правда или ложь)
        
        if (decision == true)
        {
            Console.WriteLine("Вы выбрали правильный путь!");
        }
        else if (decision == false)
        {
            Console.WriteLine("Вы ошиблись. Путь оказался неверным.");
        }
        else
        {
            Console.WriteLine("Ваш выбор пока неясен. Продолжайте исследовать.");
        }
    }

    static bool? GetPlayerDecision()
    {
        Random rand = new Random();
        int choice = rand.Next(0, 3); // 0 - ложь, 1 - правда, 2 - неизвестно

        return choice switch
        {
            0 => false,   // Ложь
            1 => true,    // Правда
            _ => null,    // Неизвестно
        };
    }
}

```
