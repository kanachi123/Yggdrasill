# **OctTree** - это дервевидная структура похожая на QuadTree  

```

Тип древовидной структуры данных, в которой у каждого внутреннего узла ровно восемь «потомков». Восьмеричные деревья чаще всего используются для разделения трёхмерного пространства, рекурсивно разделяя его на восемь ячеек. Октодеревья являются трёхмерными аналогами квадродеревьев. Англоязычное название «octree» сформировано из oct + tree и обычно пишется как «octree», а не «octtree»

```

![f](https://upload.wikimedia.org/wikipedia/commons/3/35/Octree2.png?uselang=ru)
```
Идея:

берём объём пространства;
делим его на 8 частей (октантов);
при необходимости каждую часть снова делим на 8;
получается дерево.

Почему 8?
Потому что в 3D у центра есть:

левый/правый,
верхний/нижний,
передний/задний

→ 2×2×2=8.
```
![d](https://upload.wikimedia.org/wikipedia/commons/5/51/Esfera_octree.png)

```
PR-октодерево:

деление идёт относительно реальной точки, хранящейся в узле;
центр зависит от данных;
может работать даже с бесконечным пространством
```
```
MX-октодерево:

деление идёт относительно центра текущей области;
центр вычисляется автоматически;
требует заранее ограниченного пространства.

```


```
Почему октодерево не k-d дерево:

k-d дерево делит пространство по одной оси за раз и имеет 2 потомка;
октодерево делит сразу по трём осям и имеет 8 потомков.
```
