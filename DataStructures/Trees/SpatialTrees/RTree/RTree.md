# **RTree**
```

R-дерево - древовидная структура данных которая похожа на BTree,но используется для организации доступа к пространственным данным.Например найти ближаих покемонов от моей геолокации.

```

## **в чем идея**

```

она разбивает пространство на иерархически вложенные прямоугольники которые вероятно пересекаются

# **В чем прикол**
```

это структура данных для пространственного индексирования.Она нужна чтобы быстро искать объекты на плоскости либо в многомерном пространстве

```
Например  

точки на карте,
прямоугольники,
полигоны,
зоны пересечения,
ближайшие объекты

R-tree активно используется в:

GIS-системах(географические слои например все здания,реки,деревья итд),
игровых движках,
CAD(моделирование чего то инженерного),
базах данных вроде PostGIS и SQLite (модуль Spatialite),
геопоиске.

## **в чем идея**

```
RTree индексирует области в которых хранятся данные.

Каждый объект описывается через MBR(не путать с дисковым master boot record) Minimum Bounding Rectangle)


Алгоритм:

Проверяем пересечение Q с root children.
Идем только в пересекающиеся ветки.
Остальные поддеревья пропускаем.
где Q это область в котором ищем
Это дает огромный выигрыш.

например искать погасших в конкретном городе Лейндель
мы отсекаем грув стрит оно не пересекается

Операции
1. Insert

Добавление объекта:

ищем узел, куда вставка увеличит площадь минимально,
добавляем,
если узел переполнен → split.
2. Search

Ищем:

пересечение,
containment,
nearest neighbor.
3. Delete

Удаляем объект и при необходимости перестраиваем часть дерева.

важная оговорка мы определяем макисмальный размер прямоугольника,если при вставке узел переполнен нужно делить прямоугольник
