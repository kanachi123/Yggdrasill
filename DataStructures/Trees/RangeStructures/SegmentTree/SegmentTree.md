# **SegmentTree**  

#### **Предисловие**  
Функция  
𝑓 называется ассоциативной, если:  

**𝑓(𝑎,𝑓(𝑏,𝑐))=𝑓(𝑓(𝑎,𝑏),𝑐)**  

То есть порядок скобок не важен, главное — порядок элементов  

в программировании  
***Левоассоциативность:**  
**a - b - c** обычно трактуется как **(a - b) - c**  
 (выполняется слева направо)*  

***Правоассоциативность:** Операции присваивания часто правоассоциативны: **a = b = c <=> a = (b = c)***  

Фенвик:

✔ быстрый  
✔ компактный  
❌ работает ТОЛЬКО с операциями типа:  

сумма  
иногда XOR / минимум (с костылями)  

❌ плохо поддерживает:  

максимум/минимум на отрезке с обновлениями  
сложные ассоциативные операции  
присваивание на отрезке  
любые «непрефиксные» запросы  

дерево отрезков:  

✔ универсальное  
✔ поддерживает любые операции, если они ассоциативны  
✔ можно:  

min / max  
sum  
gcd  
количество  
матрицы, структуры, кастомные объекты  
lazy propagation (очень важно)  

тоже O(log n), но гибче  

### ***Определение***  

```
Дерево отрезков - это структура данных которая помогает найти значенине некоторой ассоциативной функции f на отрезке a[i],a[i+1],...,a[j] при O(log n)

чаще в качестве f берутся функции суммы,произведения,максимум и минимум  
```

```
неравномерно это плохо губины разные
            [0,4]
         /         \
     [0,2]         [3,4]
     /   \         /   \
  [0,1] [2,2]   [3,3] [4,4]
   /   \
[0,0] [1,1]

```

*Мы берём размер N = 2^k, чтобы любой отрезок в дереве можно было рекурсивно делить пополам без остатка до единичных отрезков*

```
Если длина отрезка — степень двойки:
N = 8
то происходит идеально:
8 → 4 + 4  
4 → 2 + 2  
2 → 1 + 1  

Если НЕ степень двойки
N = 5

то происходит неидеально:
5 → 3 + 2  (если рекурсино разделим еще на раз 3 не делится на два целочисленно в итоге лишний единичный отрезок что добавляет глубину на раз и дерево становится неравномерным) проще говоря
*если делить неидеально, то левое поддерево может стать глубже правого — и как будто появляется лишняя глубина* дерево станет несимметричным
```

1. Удобная верхняя граница памяти

Мы хотим гарантировать:  
размер дерева ≤ 4n

2. Удобная структура индексов

Когда дерево “полное” (на уровне 2^k):

легко считать потомков: 2v, 2v+1
легко оценивать высоту: k

Мы выбираем k НЕ ради симметрии формы

а потому что:

после 
k делений пополам мы гарантированно дойдём до единичных сегментов

есть рост: 
$2^k$  
есть обратный лог: 
$log_{2}n$

k = ceil($log_{2}n$)

*Но важный момент
Мы НЕ “исправляем дерево”
мы просто выбираем:
ближайший уровень, где покрытие массива удобно реализуется как полное дерево*

```
выбирают 
k = ⌊log₂ n + 1(берем на запас)⌋, потому что это минимальная высота, при которой можно представить массив как полное бинарное разбиение на интервалы
```

void build()
{
    build(1, 0, (1 << h) - 1);//[0,2^h - 1] перегрузку функции для дефолтного значения и выбора отрезка,битшифт на 1 в лево это умножение на два если h раз это 1<<h <=> 2^h
}
/* 
v это номер узла в массиве в котором храним дерево
[L, R] отрезок для построения сегментного дерева
 */
void build(int v, int L, int R)
{
    //случай листа просто берем из исходной копируем для сегмента [1,1],[2,2],...,[2^k-1,2^k-1]
    if (L == R){
        b[v] = a[L];
    }
    else {
        //рекурсивно делим дерево на отрезки как яблоко ножом так чтобы всем досталось поровну
        int C = (L + R) / 2;//мидпоинт промежутка
        build(v * 2, L, C);//левая часть(потомок)
        build(v * 2 + 1, C + 1, R);//правая часть(потомок)
        b[v] = b[v * 2] + b[v * 2 + 1];//храним сумму левого и правого потомков(можно и другую ассоциативную зависимо от цели например в численных методах)
    }
}
