# FenwickTree

Дерево Фенвика (двоичное индексированное дерево, англ. Fenwick tree, binary indexed tree, BIT) — структура данных, позволяющая быстро изменять значения в массиве и находить некоторые функции от элементов массива. Впервые описано Рябко Б.Я. в 1989 году(вкпедия 
https://ru.wikipedia.org/wiki/%D0%94%D0%B5%D1%80%D0%B5%D0%B2%D0%BE_%D0%A4%D0%B5%D0%BD%D0%B2%D0%B8%D0%BA%D0%B0)  

1. Fenwick Tree (Binary Indexed Tree)
Это структура данных, реализованная через массив.
Используется для префиксных сумм и обновлений.
“Дерево” — это логическая структура, а не реально связанные узлы.
Часто называют алгоритмом Фенвика, но правильнее — это структура с алгоритмом работы над ней.

Пример использования: HP персонажей, суммарный урон по диапазону, частые обновления массива.

2. Алгоритм Фенвика (Fenwick’s algorithm)
Иногда под этим имеют в виду конкретный способ работы с массивом, который позволяет строить BIT и делать суммирование/обновления.
То есть алгоритм — это инструкции, а дерево — это структура данных, где эти инструкции применяются.

напомню закон больших чисел  
*при многократном повторении одного и того же эксперимента среднее значение результатов становится близким к математическому ожиданию (среднему значению) и почти не зависит от случая*  

```
Задача  
У нас много игроков, у каждого есть:
kills[i] — количество убийств;
games[i] — количество сыгранных матчей.
Ранг игрока = среднее количество убийств за игру = kills[i] / games[i].

Цель:

Быстро подсчитать суммарные показатели за диапазон игроков (например, топ‑10, топ‑100) или ранжировать игроков.
Проверить, как средний результат стабилизируется при увеличении количества игр (ЗБЧ)

Если храним массив kills[i] и массив games[i], то для быстрого подсчета сумм на диапазоне можно использовать Fenwick Tree:
FT_kills.range_sum(l, r) → суммарные убийства на диапазоне
FT_games.range_sum(l, r) → суммарное количество игр на диапазоне
Средний ранг на диапазоне = sum_kills / sum_games
Fenwick Tree позволяет динамически обновлять значения: игрок сыграл новую игру или сделал убийство → update(i, delta).
Закон больших чисел:
Чем больше игр сыграл игрок, тем ближе его средний ранг к истинному среднему урона/эффективности.
Для игроков с мало игр среднее сильно колеблется, для игроков с сотнями игр — стабилизируется.

```

```python

class FenwickTree:
    def __init__(self, n):
        self.n = n
        self.tree = [0]*(n+1)

    def update(self, i, delta):
        while i <= self.n:
            self.tree[i] += delta
            i += i & -i

    def query(self, i):
        res = 0
        while i > 0:
            res += self.tree[i]
            i -= i & -i
        return res

    def range_sum(self, l, r):
        return self.query(r) - self.query(l-1)

# Пример с игроками
players = 5
kills = [10, 50, 30, 0, 70]     # убийства
games = [2, 25, 10, 1, 40]      # сыгранные игры

FT_kills = FenwickTree(players)
FT_games = FenwickTree(players)

for i in range(players):
    FT_kills.update(i+1, kills[i])
    FT_games.update(i+1, games[i])

# Считаем средний ранг топ-3 игроков
l, r = 1, 3
sum_kills = FT_kills.range_sum(l, r)
sum_games = FT_games.range_sum(l, r)
average_rank = sum_kills / sum_games
print(f"Средний ранг игроков {l}-{r}: {average_rank:.2f}")

# Если каждый игрок сыграл больше игр (ЗБЧ), средний ранг будет стабилизироваться
```
