# ***Stack*** 

## ***Стек это структура данных или абстрактный тип данных представляющий собой список элементов,организованных по принципу LIFO(last in - first out,последним пришел - первым ушел)*** 

### **Это можно сравнить со стопкой тарелок,чтобы взять самую первую тарелку нужно взять последную,потом предпоследную,потом предпредпоследную...и так до этой тарелки которую мы поставили первым по очереди.**

### ***Адресация и ссылки*** 

## *Адресация Памяти это метод идентификации и обращения к данным в озу через уникальные числовые адреса ячеек.* 
## *Она преобразует логические запросы программы в физические адреса на шине* 
## *Память организована как линейный массив байтов,где каждый байт имеет свой адрес* 

## *Когда мы говорим о стеке в контексте ссылок, важно понять, что это часто связано с указателями. Каждый элемент стека может быть связан с предыдущим через адреса памяти. Например, если стек реализован как связанный список, то каждый элемент будет содержать указатель на предыдущий элемент, что позволяет восстанавливать цепочку элементов.* 

## *В случае с массивом или непрерывным блоком памяти адресация будет линейной. Однако, в случае связанного списка каждый узел указывает на следующий, и это уже "ссылка" на другой элемент в памяти.* 

# **немного асемблера** 
#### *SP -  это специальный регистор,который хранить адрес вершины стека(указатель на верхную тарелку),т.е. на последний элемент в стеке* 
### *PUSH (положить данные на стек): Когда ты вызываешь команду PUSH, значение из регистра или памяти записывается в стек, и регистр SP уменьшается* 
### *POP (извлечь данные из стека): Когда вызывается команда POP, данные извлекаются с вершины стека, и регистр SP увеличивается, указывая на следующий элемент стека* 

[0x100] <---SP(Stack Pointer)  
[0x104]  
[0x108]  
[0x10C] <---top  
### **в памяти адресация идет снизу вверх(наоборот)** 

### **Если выполняется команда PUSH 0x100, то регистр SP уменьшится и теперь будет указывать на следующую пустую ячейку. После этого, данные будут записаны в эту ячейку(смотреть вниз)**

            |

## *другой пример стека это вызов констукторов и деструкторов*

## *Иерархия конструкторов и деструкторов:*

## *Конструкторы вызываются в обратном порядке (от самых верхних классов до самых нижних).*

## *Деструкторы вызываются в обратном порядке (от самых нижних классов до самых верхних).*
## *вот здесь вступает в работу уже bp* 

```asm
;Когда функция вызывается, и ты сохраняешь значения в стеке, это выглядит ;примерно так:

push ebp          ; Сохраняем старое значение BP
mov ebp, esp      ; Устанавливаем новый BP для текущего фрейма
sub esp, 16       ; Выделяем пространство для локальных переменных

;Теперь все локальные переменные в функции будут расположены относительно ebp:

mov eax, [ebp+8]  ; Доступ к параметру функции
mov ecx, [ebp-4]  ; Доступ к локальной переменной

;это похоже на *(array + step)

```

## **SP (Stack Pointer) — это указатель, который сдвигается по мере того, как добавляются новые элементы в стек. Если ты добавляешь новые данные (например, локальные переменные в функции), SP будет сдвигаться вниз, указывая на верх стека.**

## **BP (Base Pointer) — это фиксированный указатель, который указывает на начало фрейма стека для текущей функции. В отличие от SP, который изменяется по мере добавления и удаления данных, BP остается фиксированным и помогает обращаться к локальным переменным и параметрам функции.**

### аналогия
### *Что общего с *(array + step)?*

#### *Сдвиг указателя:*

#### *SP и BP можно рассматривать как указатели, которые сдвигаются по памяти при вызове функций или работе со стеком.*

#### *SP меняется динамически в ответ на операции push и pop (прибавляется или вычитается размер данных).*

#### *BP фиксируется на начальной точке фрейма функции и используется для доступа к локальным переменным.*

#### *Получение данных через сдвиг:*

#### *В стеке мы также обращаемся к данным через сдвиг указателей. Например, если у нас есть фрейм функции, то доступ к локальным переменным осуществляется через смещение от BP.*

#### *Точно так же, как *(array + 5) даёт доступ к элементу массива, мы можем использовать смещения относительно BP, чтобы получить доступ к локальным переменным или параметрам.*

```cpp
#include <iostream>

class A {
public:
    A() { std::cout << "Конструктор A\n"; }
    ~A() { std::cout << "Деструктор A\n"; }
};

class B : public A {
public:
    B() { std::cout << "Конструктор B\n"; }
    ~B() { std::cout << "Деструктор B\n"; }
};

class C : public B {
public:
    C() { std::cout << "Конструктор C\n"; }
    ~C() { std::cout << "Деструктор C\n"; }
};//каскадное наследование

int main() {
    std::cout << "Создание объекта C:\n";
    C c; // Создание объекта C, который вызывает конструкторы в иерархии
    std::cout << "Объект C создан.\n";
    return 0;  // Уничтожение объекта C, вызов деструкторов в обратном порядке
}

/*


Создание объекта C:
Конструктор C
Конструктор B
Конструктор A
Объект C создан.
Деструктор C
Деструктор B
Деструктор A


*/

```

```asm

BP     ->  адрес возврата
       ->  параметр a

       ->  параметр c
       ->  локальная переменная d

       mov eax, [ebp + 8]  ; eax = параметр a (первый параметр перед BP)
       mov ebx, [ebp + 12] ; ebx = параметр b
       mov ecx, [ebp + 16] ; ecx = параметр c
       add eax, ebx         ; eax = a + b
       add eax, ecx         ; eax = a + b + c
```
