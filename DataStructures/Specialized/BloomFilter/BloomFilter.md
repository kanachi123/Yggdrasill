### *чтобы понять почему "да" не всегда работает,нужно разобрать конструкцию Фильтра*

### *Фильтр Блума мы храним как битовый массив из m бит,если в ней нет элемента то есть хранит пустое множество все m бит обнулены,пользовтель определяет для каждого элемента независимые хеш функции $h_1$,...,$h_k$,каждая из которых определяет индекс под которым будет 1 если все индексы для данного объекта 1,вероятно он есть:*  
### *где k количество хеш функции для объекта*  
### *$h_1$=0  $h_2$=2 $h_3$=6 $h_{m-1}$*  
### *[0][1][2][3][4][5][6][7][8]* это индексы по хешам  
### *[1][0][1][0][0][0][1][0][0]* это сам фильтр блума  

## *в чем проблема?*  
### *если количество хещ функции мало то разброс значением может быть таким что будет колизия,сейчас пример*  
bit[h1(x)] = 1  
bit[h2(x)] = 1  
...   
bit[hk(x)] = 1   

### *Несколько хеш-функций уменьшают вероятность*  
### *случайного совпадения всех битов,*  
### *необходимых для принадлежности элемента.*  

Если вероятность того что один бит уже равен 1 — p,  
то вероятность совпадения всех k:  
$p^k$  

Чем больше элементов добавляем, тем больше битов становятся 1, и:  
p → растет  
И в какой-то момент фильтр превращается почти в:  
111111111111111111111  

вероятность ошибки Bloom Filter имеет вид  
$p = (1 - e^{-kn/m})^{k}$
